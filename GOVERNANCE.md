# Project governance

## Project leadership

Rewind Cup is maintained by David Oti. The maintainer is responsible for the
project direction, release decisions, moderation, licensing decisions, and the
official deployment at https://rewindcup.com.

## How decisions are made

Routine fixes and small enhancements are decided through pull-request review.
Larger changes should begin with an issue or discussion describing the user
problem, alternatives, data implications, accessibility impact, and migration
cost.

The maintainer seeks community input, but retains final responsibility for:

- historical-data standards and disputed interpretations;
- user experience and accessibility requirements;
- dependencies and platform architecture;
- security and responsible-disclosure decisions;
- use of the Rewind Cup name and official brand assets;
- production deployment and domain administration.

## Contributors and maintainers

Contributors earn trust through accurate, well-sourced, focused work and
constructive review. Regular contributors may be invited to triage issues or
review pull requests. Repository permissions are granted explicitly and can
be changed when responsibilities or availability change.

No contribution automatically grants deployment access, ownership of the
official domain, or authority to speak for the project.

## Changes to governance

Governance changes should be proposed in a public issue or pull request except
where privacy or security requires private discussion. Material changes will
be documented in this file.
